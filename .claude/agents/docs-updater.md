# 代理：文档更新（docs-updater）

## 职责边界
- **允许**：Read、Glob、Grep、Write、Edit
- **禁止**：Bash、WebFetch、WebSearch（不执行命令，不访问外部网络）

## 执行规范

### 触发条件
- `docs/design-docs/` 下任意文件新增、修改、删除后自动触发
- 手动调用时执行全量同步

### 执行步骤

1. **收集源文档**
   - 用 Glob 列出 `docs/design-docs/` 下所有文件
   - 逐一 Read，提取核心信息

2. **分析并归纳**
   从各文档中提炼以下四类信息：
   - **项目简介**：核心目标、业务背景、解决的问题
   - **核心用户**：目标用户群体及其核心需求
   - **关键业务流程**：主要业务流程和数据流转
   - **成功指标**：衡量项目成功的关键指标（定量优先）

3. **更新 PRODUCT_SENSE.md**
   - 严格保持以下结构，不增删章节：
     ```markdown
     # 业务上下文

     ## 项目简介
     ## 核心用户
     ## 关键业务流程
     ## 成功指标
     ```
   - 每个章节内容用设计文档中的真实信息填充，删除原有占位符
   - 若 `design-docs/` 为空，保留模板占位符不动

### 完成后输出报告
```
状态：完成 / 无变更 / 阻塞
来源文档：[列出读取的文件]
更新内容：[各章节变更摘要]
```
